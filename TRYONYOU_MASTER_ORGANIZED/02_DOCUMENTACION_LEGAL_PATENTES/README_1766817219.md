# Integración con Google Drive

## Visión General

Este directorio describe el rol de **Google Drive** como nuestra plataforma centralizada para la colaboración en documentos y la gestión de archivos de oficina dentro del proyecto **TRYONYOU – ABVETOS**.

Mientras que el código fuente reside en GitHub y los datos estructurados en bases de datos, Google Drive es el espacio de trabajo para todos los activos no técnicos y documentos colaborativos que impulsan la estrategia, la gestión y la comunicación del proyecto.

## Casos de Uso Clave

### 1. Colaboración en Tiempo Real
- **Documentos (`.gdoc`)**: Para la redacción colaborativa de planes de negocio, comunicados de prensa, y documentación técnica inicial.
- **Hojas de Cálculo (`.gsheet`)**: Para la gestión de presupuestos, análisis de mercado, y seguimiento de KPIs.
- **Presentaciones (`.gslides`)**: Para la creación de presentaciones para inversores, demos de producto y reuniones de equipo.

### 2. Repositorio Central de Conocimiento
Google Drive actúa como la "fuente de verdad" para todos los documentos de negocio. Esto asegura que todo el equipo, desde los ingenieros hasta el equipo de marketing, trabaje con la información más actualizada.

### 3. Control de Versiones y Comentarios
El historial de versiones y la funcionalidad de comentarios de Google Drive son cruciales para rastrear decisiones, proporcionar feedback y mantener un registro de la evolución de la estrategia del proyecto.

## Relación con Otras Herramientas

- **NotebookLM**: Mientras que NotebookLM se usa para la *investigación y síntesis* de conocimiento, Google Drive se usa para la *creación y colaboración* en los documentos finales que resultan de esa investigación.
- **Cloud Drive (Asset Management)**: Google Drive no se utiliza para almacenar activos de producción pesados (como vídeos o escaneos 3D). Esos se gestionan en un sistema de almacenamiento en la nube dedicado (ver `docs/asset-management/README.md`).

Al mantener esta separación clara, aseguramos un flujo de trabajo eficiente y organizado, donde cada herramienta se utiliza para su propósito óptimo.
